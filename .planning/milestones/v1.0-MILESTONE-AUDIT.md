## Known Tech Debt (Non-Blocking)

**`bundle-build.test.ts > storefront bundle size is < 250KB minified (D-14)` — FAILING.**

Bundle size after Phase 8.1's real composition: **1420KB**. Previous size (placeholder JSX): 197KB. The 250KB cap was set in Phase 6 when StorefrontDrawer was a placeholder. Wiring the real chat-ui surface + DbBacked stores into the storefront bundle pulled in heavy dependencies.

**Resolution paths for a follow-up phase:**
1. Code-split `extensions/chat-drawer/src/entry.tsx` to lazy-load `ChatPane`/`HistoryPanel`/`SavedProductsPanel` on FAB click.
2. Strip dev-only imports (motion, dev tooling) from the storefront build.
3. Relax the 250KB cap with explicit PROJECT.md approval (it was a speculative budget, not a measured requirement).

**Why this is tech debt, not a milestone blocker:** Bundle size is a delivery-quality metric. The storefront chat functionally works (loads, drawers, composes real panes, chats). A 1.4MB bundle is acceptable for first ship; optimization is a fast-follow.
