<objective>
Rewrite `prisma/schema.prisma` and produce the destructive migration SQL that makes the entire product data model multi-tenant by composite key (FND-01, D-01, D-02, D-04). This plan only authors the schema + migration file — Plan 05 applies the push (`bunx prisma migrate dev`) and Plan 06 implements the repository against the regenerated client.

Purpose: Make cross-tenant data leakage structurally impossible at the database level. Every product/variant/image/option/embedding row carries `shop` NOT NULL. Children reference their parent through `(productShop, productId)` → `Product(shop, id)` so a child cannot be linked to a parent in a different shop. The application-layer signature enforcement in Plan 06 stacks on top of this DB-level guarantee.
