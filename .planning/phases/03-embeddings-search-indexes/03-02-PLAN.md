<objective>
Implement the `buildSearchableText(product)` helper at `services/search/searchableText.ts` — the single source of truth for what text gets embedded by both the sync batch step (`embed-batch-${cursor}`, plan 03-06) and the webhook handler (plan 03-07). Decision D-03 locks the format; this plan implements it and fills the test scaffolds from plan 03-01.

Purpose: Without a single helper, the sync path and webhook path could embed different text shapes for the "same" product, silently destroying search recall. Putting both call sites behind one function eliminates this drift.

Output: One small pure-function module (≤40 lines) + a test file with ≥5 passing assertions (replacing the it.todo scaffolds from plan 03-01).
</objective>
