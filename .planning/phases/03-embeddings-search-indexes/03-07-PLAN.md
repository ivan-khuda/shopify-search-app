<objective>
Wire EmbeddingService.embedAndStore into the existing Shopify webhook handler for `products/create | products/update`, convert plan 03-01's it.todo entries to real assertions, and document the new env vars + workflow in CLAUDE.md. This delivers the real-time arm of Phase 3: when a merchant edits a product in Shopify Admin, the embedding is regenerated synchronously within the same webhook handler.

Purpose: Without webhook re-embedding, search results would silently go stale every time a merchant updates a product description. D-02 chose inline synchronous embedding for simplicity — the AI Gateway call adds ~300ms to the webhook, well under Shopify's 5s budget (Pitfall 3 mitigated via try/catch + log + 200 on embed failure).

Output: One modified route, one modified test file, one documentation update.
</objective>
