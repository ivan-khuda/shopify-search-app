<objective>
Ship the `withHnswIterativeScan(callback)` helper at `lib/db/hnsw.ts` and convert plan 03-01's it.todo scaffolds in `lib/db/__tests__/hnsw.test.ts` to real assertions. This helper is the EMB-06 contract surface — Phase 4's SearchService consumes it for every shop-scoped pgvector query.

Purpose: Without `SET LOCAL hnsw.iterative_scan = 'relaxed_order'`, Postgres can silently bypass the HNSW index when the planner thinks a sequential scan beats it (especially common on shop-filtered queries). Centralising the SET LOCAL inside a transaction-callback wrapper means the array form of `$transaction` (which breaks SET LOCAL across connections — Pitfall 1) cannot be accidentally used.

Output: One small helper module (≤30 lines) + a unit test file replacing the it.todo scaffolds.
