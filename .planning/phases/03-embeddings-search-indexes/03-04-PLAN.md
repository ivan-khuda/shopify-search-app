From the `ai` package v6.0.77 (verified by RESEARCH.md §"Pattern 1"):
```typescript
import { embed, embedMany } from 'ai';

// embed:
const { embedding } = await embed({
  model: 'openai/text-embedding-3-small',  // string form auto-routes via bundled gateway provider
  value: text,
  maxRetries: 2,
});
// embedding: number[] of length 1536

// embedMany:
const { embeddings } = await embedMany({
  model: 'openai/text-embedding-3-small',
  values: texts,
  maxRetries: 2,
});
// embeddings: number[][] of length texts.length, each length 1536
```

AI Gateway auth: env var `AI_GATEWAY_API_KEY` is read implicitly by the bundled `gateway` provider (no explicit config). Plan 03-01 already added this to `.env.example` and the human checkpoint confirmed it in local `.env`.

Target service exports (per D-09):
```typescript
export const EMBEDDING_MODEL = 'openai/text-embedding-3-small' as const;
export const EMBEDDING_DIMENSIONS = 1536 as const;

export interface EmbedBatchResult {
  ok: Array<{ index: number; vector: number[] }>;
  failed: Array<{ index: number; message: string }>;
}

export async function embed(text: string): Promise<number[]>;
export async function embedBatch(texts: string[]): Promise<EmbedBatchResult>;
export async function embedAndStore(shop: string, productId: number, text: string): Promise<void>;
```

Note the local function `embed` shadows the imported `embed` from `ai`; rename the import to `embedSdk` to avoid collision.
