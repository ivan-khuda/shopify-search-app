<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → localStorage | StorefrontAdapter + LocalStorage*Store write user-scoped data; multi-tenant key namespacing is the protection. |
| App Bridge runtime → adapter | Token never logged; never cached; never embedded in thrown error messages. |
| barrel → adapters | Adapters live as sub-paths NOT re-exported from barrel (Plan 04 enforces this) — prevents App Bridge from being pulled into storefront bundle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Information Disclosure | LocalStorageHistoryStore + LocalStorageSavedProductsStore constructors | mitigate | Constructor throws `Error('LocalStorage*Store requires a non-empty scope')` when scope is empty — prevents shop-less key collision across tenants in admin context. Plan 01 Task 3 Tests 1+2 enforce. |
| T-5-AC | Repudiation | EmbeddedAdapter `getAuthHeaders` | mitigate | No module-level cache of resolved header. `shopify.idToken()` (short-lived JWT, ~1 min TTL) called on EVERY invocation. Plan 01 Task 2 Test 4 enforces. |
| T-5-ER | Information Disclosure | EmbeddedAdapter token-leak via error message | mitigate | NO try/catch wrapping in adapter — original error propagates without token concatenation. Verified by absence of try-catch in `lib/chat-ui/adapters/embedded.ts` (grep gate). |
| T-5-LO | Information Disclosure | adapter console logging | mitigate | ZERO `console.*` calls in `lib/chat-ui/adapters/*.ts` (security review §V14 + CLAUDE.md "no secrets in logs"). Grep gate enforces. |
