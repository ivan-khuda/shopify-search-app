## What Was Built

Three coupled fixes closing the storefront chat pathway end-to-end at source-code level:

**B-2 (loader.js bundle-manifest URL):** `extensions/chat-drawer/assets/loader.js` line 45 previously fetched `/apps/smartdiscovery/storefront-manifest.json` — a path that no backend route serves. Changed to `/apps/smartdiscovery/_meta/bundle-url`, which maps via App Proxy to `app/api/proxy/_meta/bundle-url/route.ts`. The `.then(r => r.json()).then(m => import(m.bundle))` chain is unchanged — backend returns the same `{ bundle: '<CDN URL>' }` shape.

**B-4 (StorefrontAdapter endpoint):** `lib/chat-ui/adapters/storefront.ts` `readonly endpoint` was `'/api/proxy/chat'` — a same-origin path on the app server, not reachable from the merchant's storefront. Changed to `'/apps/smartdiscovery/chat'`, which is the Shopify App Proxy path that Shopify HMAC-signs and forwards to `app/api/proxy/chat/route.ts`.

**W-3 (misleading test pins):** Two assertion sites in `storefront-adapter.test.ts` were asserting the wrong endpoint as "STR-08 compliance" and a comment block was encoding the bug as a contract. Both assertion sites updated to assert `/apps/smartdiscovery/chat`. The comment block replaced with an accurate explanation of App Proxy semantics.

**Deviation (Rule 1):** During Task 4's test sweep, `extensions/chat-drawer/__tests__/loader.test.ts` line 97 was asserting `loaderText.toMatch(/storefront-manifest\.json/)` — the same W-3 pattern (asserting the bug as a contract). Updated to assert `/_meta\/bundle-url/` to match the B-2 fix.
