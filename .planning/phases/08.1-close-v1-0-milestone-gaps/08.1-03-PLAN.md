<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Storefront DOM (untrusted runtime) → DbBacked stores → App Proxy backend | Storefront-supplied props (`shop`, `visitorId`, `customerId`) are passed into the DbBacked store hooks which call App Proxy endpoints — the App Proxy HMAC verification (STR-04, Phase 6) is the trust gate; this plan does NOT bypass it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.1.03-01 | Tampering | visitorId / customerId props from storefront | accept | The DbBacked stores POST these values to `/apps/smartdiscovery/conversations` and `/apps/smartdiscovery/saved-products` route handlers, which Shopify HMAC-signs and the backend handlers re-validate per Phase 6 STR-04. Hostile values fail the HMAC check; benign-but-spoofed values land in visitor-scoped rows that are still shop-bound (multi-tenant isolation preserved). |
| T-08.1.03-02 | Information Disclosure | DbBacked store instantiation in DrawerBody | mitigate | The `DrawerBody` extraction guarantees stores never instantiate when visitorId is empty (Pitfall 3 — `DbBackedHistoryStore` throws on empty visitorId at constructor time). This prevents a render-time exception that could expose stack-trace data in user-facing error overlays. |
| T-08.1.03-03 | Denial of Service | Open drawer initial mount triggers two `void store.refresh()` calls | accept | Each store's `refresh()` is one App Proxy GET; rate is per-FAB-open. The CAP-01..03 cap (Phase 8) only applies to chat completions, not history/saved reads. Storefront read cost is bounded by user clicks. |
| T-08.1.03-04 | Spoofing | StorefrontAdapter (used by ChatPane) | accept | StorefrontAdapter does NOT call `getAuthHeaders()` with any bearer (returns `{}`); identity is body-bound `visitor_id` (+ optional `customer_id`). Shopify App Proxy HMAC verification at `/api/proxy/chat` is the per-request authentication; unchanged by this plan. |
</threat_model>
