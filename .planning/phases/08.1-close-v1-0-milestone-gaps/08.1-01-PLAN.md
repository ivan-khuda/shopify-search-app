<objective>
Restore the edge auth guard for `/onboarding`, `/chat`, AND `/settings` so anonymous direct hits redirect to `/api/auth` (closes B-1 and O-2 from the v1.0 milestone audit).

Purpose: FND-03 is the precondition for every embedded admin surface; without edge auth, anonymous visitors load the React shell and never get redirected to OAuth. The audit reports this symptom, but research (RESEARCH.md §"Common Pitfalls" Pitfall 1, Open Question O-1) surfaced that Next.js 16.1.6 RENAMED `middleware.ts` → `proxy.ts` — the audit's B-1 narrative may be inverted on v16. This plan starts with a curl probe that determines empirically which file convention this project's Next.js version registers, then applies the minimum-viable fix.

Output: Either (a) `proxy.ts` is verified to be registered correctly and stays put with an added `/settings/:path*` matcher entry, OR (b) `proxy.ts` is renamed to `middleware.ts` (with `export function proxy` → `export function middleware`) and the matcher is extended. Per CONTEXT D-04 + research O-1, the curl probe decides.
</objective>
