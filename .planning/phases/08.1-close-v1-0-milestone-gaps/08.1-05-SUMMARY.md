# Phase 8.1 Plan 05: Session-Resolved Shop for /chat Banner Summary

**One-liner:** Server-side Bearer-token shop resolution for the admin `/chat` banner so `displayName` matches the model `streamText` will actually use (W-2 closure).

## What Was Built

### lib/shopify/server-resolve-shop.ts (new)

A standalone Server Component helper that decodes the embedded session-token from the incoming `Authorization: Bearer <token>` request header via `next/headers`' `await headers()` (Next.js 16 async API). It:

1. Reads `Authorization` header via `await headers()`.
2. Slices the `Bearer ` prefix and calls `shopifyClient.session.decodeSessionToken(token)`.
3. Parses `payload.dest` as a URL and returns `new URL(payload.dest).hostname` only if it ends with `.myshopify.com`.
4. Returns `null` on every failure path (missing header, non-Bearer, decode throw, missing/invalid `dest`, non-myshopify hostname).

No additional exports. No `console.*`. Co-located with `auth.ts` in `lib/shopify/`.

### app/(embedded)/chat/page.tsx (updated)

- Added import of `resolveShopFromRequest` from `@/lib/shopify/server-resolve-shop`.
- Renamed `const { shop }` to `const { shop: shopFromQuery }` for clarity.
- Added `const shopFromSession = await resolveShopFromRequest()`.
- Computed `const shop = shopFromSession ?? shopFromQuery ?? ''`.
- Banner JSX is **byte-precise unchanged**: `Preview mode — using your real catalog · Model:` with em-dash U+2014 and middle-dot U+00B7.
- `<ChatShell shop={shop} />` receives the resolved (or fallen-back) shop.

3. `falls back to searchParams.shop when session-token decode throws` — confirms silent failure path.

Added `vi.mock('next/headers', ...)` and `vi.mock('@/lib/shopify/client', ...)` at the module level. Added `beforeEach` in the Phase 4 describe block defaulting `headers()` mock to return null (no Authorization header) so existing tests still use searchParams. Added `beforeEach` in W-2 block calling `vi.clearAllMocks()` + restoring `getActiveChatModel` mock.

**Test counts:** 8 Phase 4 byte-precision tests + 3 new W-2 tests = **11 total**, all pass.

## Decisions Made

1. **Helper under `lib/shopify/`** (not `app/(embedded)/_lib/`) — co-located with `auth.ts` so all session-token decode primitives group in one folder. Clean import from `@/lib/shopify/server-resolve-shop` consistent with existing `@/lib/shopify/auth` pattern.

2. **Null-coalescing fallback chain** (`shopFromSession ?? shopFromQuery ?? ''`) — simplest possible fallback. No special case needed; `resolveShopFromRequest` returns null on all failure paths, so `shopFromQuery` is transparently used.

3. **Banner byte-precision preserved** (D-14 contract from Phase 4) — only shop-resolution mechanism changes. Em-dash U+2014, middle-dot U+00B7, exact text `Preview mode — using your real catalog · Model: ` are unchanged.

4. **Phase 4 tests protected via `beforeEach` default** — Phase 4 tests now run with `headers()` returning null (no Authorization header), which causes `resolveShopFromRequest` to return null, preserving their existing `searchParams.shop` behavior.
