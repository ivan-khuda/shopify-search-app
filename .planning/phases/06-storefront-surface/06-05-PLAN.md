  <name>Task 3: Write scripts/build-storefront-bundle.ts (esbuild driver)</name>
  <files>scripts/build-storefront-bundle.ts</files>

  <behavior>
    - Calls `esbuild.build({ ... })` with: `entryPoints: ['extensions/chat-drawer/src/entry.tsx']`, `bundle: true`, `minify: true`, `format: 'esm'`, `target: 'es2020'`, `metafile: true`, `write: false`, `loader: { '.tsx': 'tsx', '.ts': 'ts', '.css': 'css' }`, `define: { 'process.env.NODE_ENV': '"production"' }`, `jsx: 'automatic'`
    - Reads output `result.outputFiles[0].text` (the compiled JS string)
    - Computes content hash via `createHash('sha256').update(code).digest('hex').slice(0, 8)`
    - Writes the file as `public/storefront-bundle.<hash>.js`
    - Reads short git SHA via `execSync('git rev-parse --short HEAD').toString().trim()`; falls back to `String(Date.now())` if git not available
    - Writes `public/storefront-manifest.json` containing exactly `{ bundle: '/storefront-bundle.<hash>.js', version: '<sha-or-timestamp>' }`
    - On error, `process.exit(1)` with the error message printed via `console.error`
    - Cleans up previous `public/storefront-bundle.*.js` files OPTIONALLY — if implemented, restrict to the storefront-bundle prefix to avoid wiping other public assets
  </behavior>
  <action>
    Use `scripts/apply-manual-indexes.ts` as the structural template (top-of-file JSDoc, `main()` + `main().catch(...)` pattern, dotenv if needed, but no DB connection here). Implement the build per RESEARCH §Bundle Build Pipeline:

    - Top-of-file JSDoc: explain the role (D-13/D-14 — produces the lazy-loaded bundle the App Embed loader fetches at first FAB click). Note that the entry file `extensions/chat-drawer/src/entry.tsx` is created by Plan 13; running this script before Plan 13 ships will fail with esbuild's `Could not resolve` error.
    - Imports: `import { build } from 'esbuild';`, `import { createHash } from 'node:crypto';`, `import { writeFileSync, readdirSync, unlinkSync, existsSync, mkdirSync } from 'node:fs';`, `import path from 'node:path';`, `import { execSync } from 'node:child_process';`
    - In `main()`:
      1. Ensure `public/` exists (`mkdirSync('public', { recursive: true })`)
      2. (Optional cleanup) Iterate `readdirSync('public')`, unlink files matching `/^storefront-bundle\\..*\\.js$/` to prevent hash drift accumulation (D-14 RESEARCH §Runtime State Inventory)
      3. Call `esbuild.build({ ... })` per the behavior config; assign to `result`
      4. `const code = result.outputFiles[0].text;`
      5. `const hash = createHash('sha256').update(code).digest('hex').slice(0, 8);`
      6. `const filename = \`storefront-bundle.\${hash}.js\`;`
      7. `writeFileSync(path.join('public', filename), code);`
      8. `let version: string; try { version = execSync('git rev-parse --short HEAD').toString().trim(); } catch { version = String(Date.now()); }`
      9. `writeFileSync('public/storefront-manifest.json', JSON.stringify({ bundle: '/' + filename, version }, null, 2));`
      10. `console.log(\`Wrote public/\${filename} (\${code.length} bytes), manifest version=\${version}\`);`
    - `main().catch(err => { console.error(err instanceof Error ? err.message : String(err)); process.exit(1); });`

    Do NOT bake any secrets via `define`. Do NOT inject `NEXT_PUBLIC_*` at this stage — the storefront bundle reads runtime config from `dataset.*` attributes on the custom element (Pattern 5 in RESEARCH). Do NOT log entry-file contents.
  </action>
