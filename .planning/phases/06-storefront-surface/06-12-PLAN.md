  <action>
    Write a self-contained IIFE script. Layout:

    ```js
    /*
     * SmartDiscovery AI loader — Phase 6 (D-13/D-15).
     * Paints FAB synchronously on theme load; lazy-loads the main React bundle
     * on first click. Designed to stay under the 100KB liquid asset cap.
     * Vanilla JS only — no TypeScript, no React, no library imports.
     */
    (function () {
      var root = document.querySelector('smartdiscovery-app');
      if (!root) return;
      var accent = root.dataset.accent || '#008060';
      var position = root.dataset.fabPosition || 'bottom_right';
      var shop = root.dataset.shop;
      var customerId = root.dataset.customerId || null;
      var loaded = false;

      // Paint FAB
      var fab = document.createElement('button');
      fab.className = 'sd-fab sd-fab--' + position;
      fab.style.setProperty('--sd-accent', accent);
      fab.setAttribute('type', 'button');
      fab.setAttribute('aria-label', 'Open SmartDiscovery AI chat');
      fab.setAttribute('aria-expanded', 'false');
      fab.setAttribute('aria-controls', 'sd-drawer');
      // Inline Sparkles SVG (lucide path) — 24px white
      fab.innerHTML = '<svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><path d="M9.937 15.5A2 2 0 0 0 8.5 14.063l-6.135-1.582a.5.5 0 0 1 0-.962L8.5 9.936A2 2 0 0 0 9.937 8.5l1.582-6.135a.5.5 0 0 1 .963 0L14.063 8.5A2 2 0 0 0 15.5 9.937l6.135 1.582a.5.5 0 0 1 0 .962L15.5 14.063a2 2 0 0 0-1.437 1.437l-1.582 6.135a.5.5 0 0 1-.963 0z"/></svg>';
      root.appendChild(fab);

      fab.addEventListener('click', function () {
        // STR-07 / Pitfall 5: check designMode at CLICK time, not at mount
        if (window.Shopify && window.Shopify.designMode === true) return;
        if (loaded) {
          if (window.smartdiscovery && typeof window.smartdiscovery.toggle === 'function') {
            window.smartdiscovery.toggle();
          }
          return;
        }
        loaded = true;
        // Paint skeleton drawer immediately (D-15)
        document.body.classList.add('sd-skeleton-open');
        // Resolve bundle URL via App Proxy (same-origin, signed by Shopify)
        fetch('/apps/smartdiscovery/_meta/bundle-url', { method: 'GET', credentials: 'omit' })
          .then(function (r) { return r.json(); })
          .then(function (m) {
            // ES module dynamic import — cross-origin works without CORS preflight for module scripts
            return import(m.bundle);
          })
          .then(function () {
            if (window.smartdiscovery && typeof window.smartdiscovery.mount === 'function') {
              window.smartdiscovery.mount({ shop: shop, customerId: customerId, accent: accent, position: position });
            }
          })
          .catch(function () {
            // On failure, remove skeleton and reset state so a retry click can try again.
            document.body.classList.remove('sd-skeleton-open');
            loaded = false;
          });
      });
    })();
    ```

    Critical constraints:
    - NO `import` statements at top of file (must be vanilla loaded via script_tag from liquid, not a module). Dynamic `import()` inside the click handler is fine — it's a runtime expression, not a module-system declaration.
    - NO `console.log` (CLAUDE.md hard constraint).
    - Use ES5-compatible syntax (`var`, no arrow functions) for maximum theme compatibility — older Shopify themes may not support ES6 reliably.
    - File size MUST stay under ~5KB minified target per D-13 (RESEARCH §Pattern 5).
    - Theme app extension total cap is 10MB, per-asset Liquid cap is 100KB (Pitfall 8) — loader.js is well under this.

    The `/apps/smartdiscovery/_meta/bundle-url` route is the resolved bundle-URL handler. Plan 13 ships it (HMAC-wrapped via withAppProxyHmac, reads `process.env.HOST` or `process.env.APP_URL`, returns `{ bundle: 'https://<host>' + manifest.bundle }`). If Plan 13's executor finds the file is missing, ship it as the first task of Plan 13.

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-30 | Spoofing | designMode bypass causing auto-open in editor | mitigate | designMode check at FAB-click time, not mount (Pitfall 5) — extension never auto-opens drawer |
| T-06-31 | Bundle integrity | Stale hash served by Vercel | accept | SRI not in V1; manifest fetched fresh on every cold load via no-cache header (loader should send cache: 'no-store' on the manifest fetch — add to Task 4 if missing); document acknowledgment |
