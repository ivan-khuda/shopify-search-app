<objective>
Implement `lib/identity/merge.ts` exporting `mergeVisitorIntoCustomer(shop, visitorId, customerId)`. Encodes D-11's exact SQL inside `prisma.$transaction`:
1. Idempotency short-circuit: if a `VisitorCustomerLink` row exists for the triple, return without writes.
2. UPDATE Conversation rows: set customerId on rows matching (shop, visitorId, customerId IS NULL).
3. INSERT SavedProduct rows with customerId set, ON CONFLICT (shop, customerId, productId) WHERE customerId IS NOT NULL DO NOTHING (dedupe).
